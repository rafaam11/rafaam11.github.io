# 케이스 스터디 자산 공개 체크리스트

케이스 스터디 페이지에 이미지/영상을 추가하기 전, 아래 항목을 모두 확인하세요.

## 공통 확인 사항

- [ ] 자산이 Digitrack 사내 승인을 받았는가? (회사 과제 자산)
- [ ] 병원·환자 데이터(CT, MRI, 환자 영상)가 포함되어 있지 않은가?
- [ ] 발주기관의 비공개 요청 사항이 없는가?
- [ ] 합성 팬텀 또는 오픈 데이터셋만 사용했는가?

## 프로젝트별 공개 가능 범위

| 프로젝트 | 스크린샷 | 영상 | 코드 | 비고 |
|---|---|---|---|---|
| 턱뼈 골절복원 | ✅ 오픈 데이터 팬텀 한정 | 확인 필요 | 논문 공개 범위 | 논문 Figure 재사용 가능 |
| 방사능 디지털트윈 | 확인 필요 | 확인 필요 | ❌ | 진행 중 과제 |
| 악교정 AR | 팬텀 한정 | 확인 필요 | ❌ | 삼성서울병원 협력 |
| Life Careverse | 확인 필요 | 확인 필요 | ❌ | 보건복지부 과제 |
| Surgical Twin | 확인 필요 | ❌ | ❌ | KERI 개발용역 |

## 디렉토리 규칙

```
assets/projects/<slug>/
├── thumb.jpg         # 썸네일 (index.html 카드 썸네일용, 400×300px 권장)
├── hero.jpg          # 케이스 스터디 히어로 이미지 (1200×630px 권장)
├── img-01.jpg        # 본문 이미지 1
├── img-02.jpg        # 본문 이미지 2
└── ...
```

대용량 영상(10MB+)은 **YouTube Unlisted 업로드 후 embed 방식**을 사용합니다 (Git 저장소 용량 절약).
